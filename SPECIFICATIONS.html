1. Architecture overview

A shared Kubernetes cluster + a shared Vault. Each client gets a fully isolated stack in its own namespace tenant-<client>. Only the backend is public; kripfs and the datastores are internal.

7. Application environment — shared env used by the backend

The backend loads all env via envFrom: a ConfigMap (non-secret) + a Secret (materialized from Vault). Source key: cfg=ConfigMap · V:tenant=per-tenant Vault · V:shared=shared Vault · dns=in-namespace service.
